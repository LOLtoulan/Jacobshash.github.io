集合总结

集合中包含两大集合：Collection和Map

Collection

  1. List

    • ArrayList
      • 底层结构是数组，查询快，增删慢，线程不安全，效率高
        • 常用方法：
          1. add();增加元素
          2. remove();删除元素
          3. isEmpty();判断集合是否为空
          4. toArray();将链表转换为数组
          5. set();替换
          6. get();获取指定元素
    • Vector
      • 底层结构是数组，查询快，增删慢，线程安全，效率低
        • 特有方法：
          1. addElements();同add()
          2. elementAt();返回指定位置索引
        • 说明：一般不用。或者说，基本不用
    • LinkedList
      • 底层结构是链表，查询慢，增删快，线程不安全，效率高
        • 特有方法
          1. addFirst()
          2. addLast()
          3. getFirst()
          4. getLast()
          5. peek()获取但不移出此列表的头(第一个元素)
          6. poll()获取并移除此列表的头
          7. pop()弹出列表头
          8. push();将元素推入集合中
          9. removeFirst()
          10. removeLast()

  2. Set（唯一，无序）

    • HashSet

      • 底层是hash表，是一个元素为链表的数组
        • 通过查看源码我们发现，HashSet的add方法是通过new HashSet时，通过无参构造创建HashMap对象，通过hashmap的put方法来达到元素唯一性。
      • 数据的唯一性依赖于HashCode和equals方法，可自动生成。
        • 先判断HashCode是否一样
          • 一样，继续比较equals，看返回值，
            • true：相同不添加
            • false：不相同，添加
          • 不一样，直接添加
      • LinkedHashSet集合
        • 此集合底层是hash表和链表，Hash表确保元素的唯一，链表确保集合的有序

    • TreeSet（可按照一定的规则排序，具体实现与构造器有关）

      • 底层时二叉树（红黑树）

    • 如何保证元素的唯一性，红黑树（元素相同则无法添加进集合中）

      • 自然排序（元素具备具备比较性）

        • 无参构造排序是因为例如Integer，String，Character都已经实现Comparable接口，是因为：此接口强行对实现它的每个类的对象进行整体排序。这种排序被称为类的自然排序，类的 compareTo 方法被称为它的自然比较方法。

    • 构造器排序

      • 有参构造则是通过让集合构造方法实现Comparator的实现类对象，而一般我们都不会新创建一个类专门来实例化对象，一般我们会通过匿名内部类的方法来实现。

  3. 一般我们在用集合时，尽量用ArrayList，其他集合能实现的功能ArrayList集合基本都能实现。一般不用Set集合，用也只用HashSet。（具体情况具体对待）

Map

将键映射到值的对象。一个映射不能包含重复的键；每个键最多只能映射到一个值。

Map集合的数据结构仅仅针对键有效，与值无关

  1. HashMap线程不安全，效率高
    • 和hashSet一样
    • LinkedHashMap
      • 和LinkedHashSet一样
  2. Hashtable线程安全，效率低
    • 和HashMap基本一样
  3. TreeSet
    • 和TreeSet一样
集合如何选择

​ 看需求

  • 是否是键值对象形式
    • 是：Map
      • 键是否需要排序
        • 是：HashMap
        • 否：TreeMap
      • 不知道就用HashMap
    • 否：Collection
      • 元素是否唯一：
        • 是：Set
          • 是否需要排序
            • 是：TreeSet
            • 否：HashSet
          • 不知道就用HashSet
      • 否：List
        • 安全否？
          • 安全Vector（其实也不用，用的最多的是ArrayList的加锁方式）
          • 否，增删多，查询多
            • 增删多：LinkedList
            • 查询多：ArrayList
        • 不知道就用ArrayList，不知道就用ArrayList，不知道就用ArrayList（重要的事情说三遍）

集合的常见方法及遍历

Collection：

  • add
  • remove
  • contains
  • iterator
  • size

遍历：

  • 增强for

  • 迭代器

List集合由三种遍历

  • 普通for，get

  • 增强for

  • 迭代器

Set

  • 增强for

  • 迭代器

Map:

  • put
  • remove
  • containsKey()
  • containsValue
  • keySet
  • get
  • value
  • entrySet
  • size

遍历两种：

for (HashMap<String,String> hashMap3 : arrayList) {
    Set<String> set = hashMap3.keySet();
    for (String key : set) {
         String value = hashMap3.get(key);
         System.out.println(key+"----"+value);
    }
}
