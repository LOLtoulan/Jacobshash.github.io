Collection集合

一、Collection集合的功能概述

Collection：Collection 层次结构 中的根接口。Collection 表示一组对象，这些对象也称为 collection 的元素。一些 collection 允许有重复的元素，而另一些则不允许。一些 collection 是有序的，而另一些则是无序的。JDK 不提供此接口的任何直接实现：它提供更具体的子接口（如List和Set）实现。此接口通常用来传递collection，并在需要最大普遍性的地方操作这些collection。

Collection的基本功能

增加：

  1. boolean add(Object obj) :添加一个元素
  2. boolean addAll(Collection c) : 添加一个集合的元素

删除：

  1. void clear() :移出所有元素
  2. boolean remove(Obejct o);移出一个元素
  3. boolean removeAll(Collection c);一处一个集合的元素

判断：

  1. boolean contains(Obejct o);判断集合中是否包含指定元素
  2. boolean containsAll(Collection c);判断集合是否包含指定集合元素
  3. boolean isEmpty(); 判断集合是否为空

长度功能：

  1. int size(); 元素的个数

    面试题：数组有没有length()方法，但是有length属性，String有length()方法，Collection没有length()方法，但有size()方法

交集功能

  1. boolean retainAll(Collection c);

把集合转换为数组

  1. Object[] toArrays()

获取功能

  1. Iterator iterator() （重点）

public class CollectionDemo {
    public static void main(String[] args) {

        //因为由JDK我们知道,Collection是一个接口,不能直接实现,所以要找子实现类,但我们发现Collection
        //没有子实现类,但是子接口List中由实现类,So,我们找到了ArraysList()

        Collection collections = new ArrayList();

        //添加元素
        collections.add("String");
        collections.add(123);
        collections.add(2.1313);
        for (Object collection : collections) {
            System.out.println(collection);
        }
        System.out.println("------------------------");


        //输出集合长度
        System.out.println(collections.size());
        System.out.println("------------------------");


        //删除一个集合元素
        collections.remove(123);
        for (Object collection : collections) {
            System.out.println(collection);
        }
        System.out.println("------------------------");


        collections.add(123456789);
        for (Object collection : collections) {
            System.out.println(collection);
        }
        System.out.println("------------------------");

        //判断某个元素是否在集合中
        System.out.println(collections.contains(123));
        System.out.println(collections.contains(123456789));
        System.out.println("------------------------");

    }
}

四、用迭代器遍历集合

迭代器：
public interfaceIterator
对 collection 进行迭代的迭代器。迭代器取代了 Java Collections Framework 中的 Enumeration。迭代器与枚举有两点不同：
迭代器允许调用者利用定义良好的语义在迭代期间从迭代器所指向的 collection 移除元素。
方法名称得到了改进。
此接口是 Java Collections Framework的成员。

该接口中共有三个方法：

boolean hasNext() 如果仍有元素可以迭代，则返回 true。
E next() 返回迭代的下一个元素。
void remove()从迭代器指向的 collection 中移除迭代器返回的最后一个元素（可选操作）。
为什么迭代器不定义成一个类，而是一个接口？

​ 现在我们假设迭代器是一个类，这样我们就可以创建该类的对象，调用该类的方法对集合进行遍历，但是，Java中提供了很多的集合类，而这些集合类的数据结构是不同的，多以存储方式和遍历方式也不尽相同，所以，最终就没有定义集合类，而无论你是那种集合，自身都具有获取元素的功能，并且辅助于判断功能，这样就不会出错，，也就是说判断功能和获取功能是一个集合遍历所具备的，而每种集合又不一样，所以就不用类实现。

那么真正具体的实现类在哪里呢？

在真正的具体的子类中，以内部类形式体现的。

        //需求:创建一个Collection集合遍历集合,并获取集合中元素的长度

        Collection collections = new ArrayList();
        collections.add("snowQueen");
        collections.add("dadada");
        collections.add("89898898");

        for (Object collection : collections) {
            System.out.println(collection+"---"+collection.toString().length());
        }
        System.out.println("-----------------------");

        Iterator iterator = collections.iterator();
        while (iterator.hasNext()) {
            System.out.println(iterator.next());
        }
    }
}

练习：用集合存储学生对象，并把学生对象进行遍历。（用Iterator）
public class Student {
    private String name;
    private int age;

    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public Student() {
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    @Override
    public String toString() {
        return "Student{" +
                "name='" + name + '\'' +
                ", age=" + age +
                '}';
    }
}

public class StudentDemo {
    public static void main(String[] args) {
        Collection c = new ArrayList();
        Student s1 = new Student("东方不败",22);
        Student s2 = new Student("令狐冲",22);
        Student s3 = new Student("斋藤飞鸟",22);
        Student s4 = new Student("桥本环奈",22);
        Student s5 = new Student("郑合惠子",22);

        c.add(s1);
        c.add(s2);
        c.add(s3);
        c.add(s4);
        c.add(s5);

        Object[] obj = c.toArray();
        for (int x = 0; x < obj.length; x++) {
            Student s = (Student) obj[x];
            System.out.println(s.getName() + "---" + s.getAge());
        }

        System.out.println("---------------");

        //刚开始我们发输出的是一串地址字符串,是因为我们没有重写toString()方法
        Iterator iterator = c.iterator();
        while (iterator.hasNext()) {
            Student s = (Student)iterator.next();
            System.out.println(s.getName()+"----"+s.getAge());
        }
        //扩展，这种方法的好处是，当程序执行完时，iterator1就变成垃圾，就会被垃圾回收机制回收
        System.out.println("----------");
        for(Iterator iterator1 = c.iterator();iterator1.hasNext();){
            Student s = (Student)iterator1.next();
            System.out.println(s.getName()+"----"+s.getAge());
        }
    }
}

总结

集合的继承体系结构：

Collection

       |--List 有序，可重复

              |--ArrayList

                   底层数据是数组，查询快，增删慢
                    线程不安全，效率高

               |--Vector

                    底层数据是数组，查询快，增删慢
                   线程安全，效率低

              |--LinkedList

                   底层数据是链表，查询慢，增删快
                    线程不安全，效率高

        |--Set

               |--TreeSet

                        底层数据是红黑树，有一定规则排序，与所选的构造器有关

                        |--LinkedListSet

               |--HashSet

                        底层数据是哈希表，通过HashCode和Equals来确保元素唯一性

                        |--LinkedListSet

                                底层数据是哈希表和链表，通过HashCode和Equals来确保元素唯一性
                               由链表来确保元素有序，由hash表来确定元素唯一

针对Collection集合我们到底该怎样选呢？

分析如下：

           唯一吗？

                   不唯一：List

                           安全吗？

                               安全：Vector（其实一般需要安全也不用Vector，一般用ArrayList加同步锁的方式）

                               不安全：ArrayList   或 LinkedList

                                       查询多：ArrayList

                                       增删多：LinkedLIst

                   唯一：Set

                           排序吗？

                               排序：TreeSet

                               不排序：HashSet

如果知道需要用Set，但不知道用哪个，就选HashSet    

如果知道需要用List，但不知道用哪个，就选ArrayList

如果知道需要用集合，但不知道用哪个，就选ArrayList    



集合中常见的数据结构：

ArrayXxx：底层数据是数组查询快，增删慢

LinkedXxx：底层结构是链表，查询慢，增删快

HashXxx：底层结构是Hash表，依赖两个方法，HashCode和Equals

TreeXxx ：底层结构是二叉树，两种排序方式，自然排序和比较器排序
