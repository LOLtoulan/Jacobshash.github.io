Java小案例之实现多线程售票

题目：

某电影院有20张票，共三个售票窗口，三个窗口同时售票，售完为止

用继承Thread的方法实现多线程：

public class ThreadSleep extends Thread {

    public ThreadSleep() {
    }

    public ThreadSleep(String name) {
        super(name);
    }
    //static修饰的变量或方法只执行一次
    private static int ticket = 20;
    @Override
    public void run() {
        while (true) {
            if (ticket > 0) {
                System.out.println(getName()+"号窗口正在售第"+(ticket--)+"张票");
            }
        }
    }
}

public class ThreadSleepDemo {
    public static void main(String[] args) {
        ThreadSleep threadSleep = new ThreadSleep("1号窗");
        ThreadSleep threadSleep1 = new ThreadSleep("2号窗");
        ThreadSleep threadSleep2 = new ThreadSleep("3号窗");

        threadSleep.start();
        threadSleep1.start();
        threadSleep2.start();
    }
}

以下是两次不同的运行结果：

)

从两次运行结果我们可以看出这种方法仍存在漏洞，所以，我们要尝试新的方法，这时我们想到了通过实现Runnable接口的方法来实现这个小案例，

撸代码：

public class MyRunnable implements Runnable{

    private  int ticket = 20;
    @Override
    public void run() {
        while (true) {
            if (ticket > 0) {
                System.out.println(Thread.currentThread().getName()+"号窗口正在售第"+(ticket--)+"张票");
            }
        }
    }
}
public class MyRunnableDemo {
    public static void main(String[] args) {
        MyRunnable myRunnable = new MyRunnable();

        new Thread(myRunnable, "1号窗").start();
        new Thread(myRunnable, "2号窗").start();
        new Thread(myRunnable, "3号窗").start();
    }
}

这个结果如下：

由上面的两个案例我们可以发现，实现Runnable接口的方法更容易实现这种数据共享的多线程任务。

所以我们认为这个程序是好的，没有bug的，所以我想添点真实的场景，因为我们买票不可能是同步的，所以是有延时的，所以我们让程序sleep100ms，看看情况，

public class MyRunnable implements Runnable{

    private  int ticket = 20;
    @Override
    public void run() {
        while (true) {
            if (ticket > 0) {

                try {
                    Thread.sleep(100);

                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                System.out.println(Thread.currentThread().getName()+"口正在售第"+(ticket--)+"张票");
            }
        }
    }
}
public class MyRunnableDemo {
    public static void main(String[] args) {
        MyRunnable myRunnable = new MyRunnable();

        new Thread(myRunnable, "1号窗").start();
        new Thread(myRunnable, "2号窗").start();
        new Thread(myRunnable, "3号窗").start();
    }
}

但是当我们在运行程序时发现出现了，如下结果：

)

这是我们意想不到的，因为售票最少是0张票，且不能卖第0张票，但是我们在运行结果中看到，不仅出售了第0张票，还出售了第-1张票，看来我们的程序本身还存在问题，有结果来看，我们的程序出现了两种问题，

1.出现出售0张票的现象

2.出现出售-1张票的现象

出现这种情况的主要原因是：

1.cpu每次执行必须是一个原子性（最简单基本的操作）的操作，即这个操作不能拆分。

但ticket–并不是一个原子性操作，ticket–是先记录–以前的值

接着把ticket–，然后输出以前的值，但是当窗口输出以前的值时（窗口2来了），

此时ticket还没有变为99，所以窗口二输出100，其他的一样类推

2.随机性和延迟导致的，

出现-1票的原因：当票数为1时，窗口1进来休息，窗口2进来休息，窗口3进来休息（此过程随机出现，不可能每次都复现），然后

窗口1结束休息结束进行ticket–操作，窗口1正在出售第1张票，ticket为0，

窗口2结束休息结束进行ticket–操作，窗口1正在出售第0张票，ticket为-1，

窗口3结束休息结束进行ticket–操作，窗口1正在出售第-1张票，ticket为-2

所以会出现-1张票，和0张票

发现问题解决问题：

要想解决问题，先要知道那些问题会导致出问题：

  1. 是否为多线程程序
  2. 是否共享数据
  3. 是否有多条语句操作共享数据（注意，如果就算有共享数据但是操作数据是原子性的，那就也不会出现问题）

那么我们来分析下我们的程序有没有上面的问题

  1. 是多线程
  2. 是共享数据
  3. 是所条语句操作共享数据，且操作不是原子性的

思想：前两条无法改，所以只能改第三个，让某个线程执行的时候，其他线程不能执行，我们不会，但是Java提供了API方法，提供了同步锁机制，

synchronized (对象) {需要同步的代码}
//对象：同步锁可以解决安全的根本原因就在对象上，该对象如同锁的功能
//需要同步的代码：示代码而定

方法，所以我们就可以用Java提供的方法进行执行。

撸代码：

public class MyRunnable implements Runnable{

    private  int ticket = 200;
    //相当于对象也变成共享的了，相当于一把锁
    private Object object = new Object();
    @Override
    public void run() {
        while (true) {
            synchronized (object) {
                if (ticket > 0) {

                    try {
                        Thread.sleep(100);

                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                    System.out.println(Thread.currentThread().getName()+"口正在售第"+(ticket--)+"张票");
                }
            }
        }
    }
}

public class MyRunnableDemo {
    public static void main(String[] args) {
        MyRunnable myRunnable = new MyRunnable();

        Thread thread = new Thread(myRunnable, "1号窗");
        Thread thread1 = new Thread(myRunnable, "2号窗");
        Thread thread2 = new Thread(myRunnable, "3号窗");

        thread.start();
        thread1.start();
        thread2.start();
    }
}

程序可以正常运行，结果就不贴了。

同步锁的优点及弊端：

使用同步锁的前提：

1.多线程

2.多个线程使用的是同一锁对象

优点：解决了多线程的安全问题

弊端：当线程相当多时，因为每个线程都会去判断同步上的锁，这是很消耗资源的，无形中降低了程序的运行效率。

同步锁的格式及锁对像问题：

同步代码块的锁对象是任意对象

把同步所关键词放在方法上，同步方法的锁对象是this

把同步所关键词放在静态方法上，同步方法的锁对象是当前类的字节码文件也就是 .class 文件

线程安全的类的回顾：

StringBuffer ，Vector，HashTable，都是线程安全的，
