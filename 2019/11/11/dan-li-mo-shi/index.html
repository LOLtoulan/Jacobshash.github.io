一、单例模式的介绍

单例模式是Java中最简单的设计模式之一，这种类型的设计模式属于创建型模式，它提供了一种创建对象的最佳方式。这种模式涉及到一个单一的类，该类负责创建自己的对象，同时确保只有单个对象被创建。这个类提供了一种访问其唯一的对象的方式，可以直接访问，不需要实例化该类的对象。

应当注意的是：

  • 单例类只能有一个实例

  • 单例类必须自己创建自己的唯一实例

  • 单例类必须给所有其他对象提供这一事例

下面写一个小的demo（饿汉模式）:

public class SingleModel {
    //创建一个私有的SingleModel对象
    private static SingleModel singleModel = new SingleModel();

    //将构造函数设置为私有这样在对象调用时就不会实例化该类
    private SingleModel() {}

    //创建一个唯一获取该类对象的方法
    public static SingleModel  getSingleModel() {
        return singleModel;
    }
    //测试函数
    public void say(){
        System.out.println("hello 单例模式！！");
    }
}

主函数:

public class SingleModelDemo {
    public static void main(String[] args) {

二、单例模式的几种实现方法

2.1.1 懒汉模式(线程不安全 , 不支持多线程)

public class SingleModel {

    private static SingleModel singleModel;

    private SingleModel() {
    }

    public static SingleModel getSingleModel() {
        if (singleModel == null)
            singleModel = new SingleModel();
        return singleModel;
    }

    public void say() {
        System.out.println("hello 单例模式中的懒汉模式！！");
    }
}

主函数:

public class SingleModelDemo {

由上面代码我们可以看出懒汉模式的由来 , 在类加载时并没有直接创建对象 , 而是当程序加载时需要这个对象时才创建买这个对象 , 就好像一个人懒的饿到不行了才去找东西吃一样 .

这种是最简单的单例模式 , 其实 , 严格意义上来说 , 这并不是单例模式 , 因为没有加 synchronized 锁 , 在多线程下不能正常工作 , 也就是说 , 如果线程是单线程的 , 是按照顺序 ( 串行 ) 的方式执行代码 , 但到了多线程下 , 执行顺序就由串行 , 变为并行 , 如果控制不好就会出现许多意想不到的情况 , 这就是多线程的安全问题 , 所以应当加锁来控制线程并行引起的问题 .

如果线程数太少的话 , 并不一定会出现上面的那种形式 , 但随着线程数的增加这种方法的弊端就越来越明显 , 这还不到30个线程 , 平均运行三次就会出现 一次 , 试想一下 , 如果有3万个 , 30万个 ,这机会出现极大的概率 , 所以这种方式是线程不安全的 .

补充：

引发线程安全的因素：

1.是否为多线程

2.是否有共享数据

3.是否有多条语句操作共享数据

2.1.2 懒汉模式(线程安全 , 支持多线程)

public class SingleModel {

    private static SingleModel singleModel;

    private SingleModel() {
    }

    public static synchronized SingleModel getSingleModel() {
        if (singleModel == null)
            singleModel = new SingleModel();
        return singleModel;
    }

    public void say() {
        System.out.println("hello 单例模式中的懒汉模式！！");
    }
}

主函数:

public class SingleModelDemo {
    public static void main(String[] args) {

        //这样会报错,见下图，编译时会出错
        //SingleModel singleModel = new SingleModel();

        //通过该类的类名调用该类中的静态方法getSingleModel(),获取该类的唯一对象
        SingleModel singleModel = SingleModel.getSingleModel();
        singleModel.say();
    }
}

多线程 : 和上面的多线程一样 , 就不再写了

这样的单例模式支持多线程 , 线程安全 , 但加锁会影响效率 .

2.2.1双检锁/双重校验锁 ( DCL , 即double-checked locking (线程安全 , 支持多线程))

public class SingleModel {
    private volatile static  SingleModel singleModel = null;
    private SingleModel() {}
    public static  SingleModel getInstance() {
        if (singleModel == null){
            synchronized (SingleModel.class) {
                singleModel = new SingleModel();
            }
        }
        return singleModel;
    }
}

主函数和上面的多线程一样 , 就不再写了 , 在这段代码中 , 我们发现了一个明显的新的关键词 , “volatile” 这个关键词起到了至关重要的作用 , 要理解为什么要加上这个关键词 , 我们必须了解对象构造过程 , 程序实例化一个对象 , 其实可以分为三步 :

  • 分配内存空间

  • 初始化对象

  • 将内存空间的地址赋值给对应的引用

由于由于操作系统在执行指令时会进行重排序，所以可能导致实例化一个对象的步骤由上面的三步转变为下面的三个步骤：

  • 分配内存空间
  • 将内存空间的地址赋值给对应的引用
  • 初始化对象

如果按照上面的步骤进行，那么在多线程下就会将一个没有初始化的对象暴露出来，从而导致不可预料的后果，而加上就不会有这些顾虑了，因为 volatile关键字就是Java中提供的另一种解决可见性和有序性问题的方案。 一看到有序我们就知道，加上volatile就会使实例化对象的时候变得有序。

2.3.1 登记式/静态内部类

public class SingleModel {

    private  static class  SingleModelHolder{
        private static final SingleModel SINGLE_MODEL = new SingleModel();
    }

    private SingleModel() {
    }

    public static  SingleModel getInstance() {
        return SingleModelHolder.SINGLE_MODEL;
    }
}

主函数和上面的多线程一样 , 就不再写了 。

这种方法有DCL一样的效果，实现更加简单，但是 这种方式只适用于静态域的情况，双检锁方式可在实例域需要延迟初始化时使用。

2.4.1 枚举

public enum SingleModel{  
    INSTANCE;  
    public void whateverMethod() {  
    }  
}

这种实现方式还没有被广泛采用，但这是实现单例模式的最佳方法。它更简洁，自动支持序列化机制，绝对防止多次实例化。

经验之谈：一般情况下，不建议使用第 1 种和第 2 种懒汉方式，建议使用第 3 种饿汉方式。只有在要明确实现 lazy loading 效果时，才会使用第 5 种登记方式。如果涉及到反序列化创建对象时，可以尝试使用第 6 种枚举方式。如果有其他特殊的需求，可以考虑使用第 4 种双检锁方式。

补充

如何判断单例模式是否成功，我们知道，单例模式就是创建单一对象，所以，我们可以想到：

public class SingleModel {
    private SingleModel(){}
    private static SingleModel singleModel = new SingleModel();
    public static SingleModel getInstance() {
        return singleModel;
    }
}
public class SingleModelDemo {
    public static void main(String[] args) {
        SingleModel singleModel = SingleModel.getInstance();
        SingleModel singleModel1 = SingleModel.getInstance();
        System.out.println(singleModel==singleModel1);
    }
}

结果为：true
