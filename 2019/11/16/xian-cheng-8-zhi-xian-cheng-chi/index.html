Java中的线程池

一、线程池的介绍

程序启动一个新线程成本比较高的，因为涉及到与操作系统进行交互。而使用线程池可以很好的提高性能，尤其是程序中要创建大量生存周期很短的线程时，更应该考虑线程池。

  • 线程池里面的每一个线程代码结束后，并不会死亡，而是再次回到线程池中成为空闲状态等待下一个线程来使用
  • 在jdk5之前，我们必须手动实现线程池，但在jdk5以后Java内置线程池

如何实现线程池的代码呢？

  • 创建一个线程池对象控制要创建几个线程对象。

    public static ExecutorService new newFixedThreadPool(int nThreads)

  • 这种线程池的线程可以执行：

    可以执行Runnable对象或者Callable对象代表的线程

    写一个类实现Runnable接口。

  • 调用方法如下即可

    Future<?> submit(Runnable task)

    Future submit(Callable task)

public class MyRunnable implements Runnable {
    @Override
    public void run() {
        for (int i = 0; i <= 100; i++) {
            System.out.println(Thread.currentThread().getName()+"---"+i);
        }
    }
}
public class ExcutorsDemo {
    public static void main(String[] args) {
        ExecutorService pool = Executors.newFixedThreadPool(2);
        pool.submit(new MyRunnable());
        pool.submit(new MyRunnable());

        //shutdown()

    }
}

二、线程的第三种创建方法

线程的第三种创建方法时通过实现Callable接口来实现的，而且这种方法还有返回值，下面来看一下线程的第三种创建方法

public class MyCallable implements Callable {
    @Override
    public Object call() throws Exception {
        for (int i = 0; i <= 100; i++) {
            System.out.println(Thread.currentThread().getName()+"---"+i);
        }
        return null;
    }
}
public class MyCallableDemo {
    public static void main(String[] args) {
        ExecutorService pool = Executors.newFixedThreadPool(2);
        pool.submit(new MyCallable());
        pool.submit(new MyCallable());

       // pool.shutdown();
    }
}

如果只是这样，那么这种方法还没Runnable好用，就不必有这个方法了，但是，前面说过，这种方法可以有返回值，下面就做一个含有返回值的案例。

代码如下：

public class MyCallable implements Callable<Integer>  {

    private  Integer n ;
    private  Integer m;

    public MyCallable() {
    }
    public MyCallable(int n, int m) {
        this.n = n;
        this.m = m;
    }
    @Override
    public Integer call() throws Exception {
        int sum = 0;
        for (int i = n; i <= m; i++) {
            sum=sum+i;
        }
        return sum;
    }
}
public class MyCallableDemo {

    public static void main(String[] args) throws ExecutionException, InterruptedException {
        ExecutorService pool = Executors.newFixedThreadPool(2);

        Future future = pool.submit(new MyCallable(0,100));
        Future future1 = pool.submit(new MyCallable(0,200));

        int i = (int) future.get();
        int i1 = (int) future1.get();

        System.out.println(i);
        System.out.println(i1);

        pool.shutdown();
    }
}

总结：

线程池，在执行完代码后不会死亡，而会空闲下来等待下一个线程使用，但线程池的大小要视情况而定。

Callable实现线程的方法有返回值，如果你的程序需要用到返回值，那么这个首选，但是，如果没有返回值，尽量不要用，太麻烦了。
