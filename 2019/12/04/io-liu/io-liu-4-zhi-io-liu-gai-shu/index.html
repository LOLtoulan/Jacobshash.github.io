IO流概述

一、IO流分类

按数据流向：

  • 输入流：读入数据（相对着Java程序而言）
  • 输出流：写出数据（相对着Java程序而言）

按数据类型：

  • 字符流：输入流，输出流
  • 字节流：输入流，输出流
  • 什么情况下使用那种流呢？
    • 如果数据通过Windows自带的记事本能打开，并能读懂，就用字符流，否则用字节流，如果啥都不知道，就用字节流。

二、IO流常用基类

字节流的抽象基类：

  • InputStream
  • OutputStream

字符流的抽象基类

  • Reader
  • Writer

注：由这四个类派生出来的子类都是以其父类名作为子类名的后缀

  • 如：InputStream的子类之一，FileInputStream
  • 如：Reader的子类之一，FileReader

三、FileOutputStream方法用法

字节输出流操作步骤

  • 创建字节输出流
  • 写入数据
  • 关闭流对象，释放资源

一个小demo：

public class Myoutput {
    public static void main(String[] args) throws IOException {
        String s = "hello world ";
        //如果没有相应的文件，则创建，如有，则覆盖，就是会清空里面的数据
        FileOutputStream fileOutputStream = new FileOutputStream("b.txt");
        fileOutputStream.write(s.getBytes());
        fileOutputStream.close();
    }
}

创建字节流对象做了哪几种事？

  • 调用系统功能去创建文件
  • 创建fileOutputStream对象
  • 把fileOutputStream对象指向这个文件

为什么一定要fileOutputStream.close(); 呢？

  • 让流对象变成垃圾
  • 通知系统去释放相该文件关资源
FileOutputStream的三种write方法用法
public class Myoutput {
    public static void main(String[] args) throws IOException {

        FileOutputStream fileOutputStream = new FileOutputStream("b.txt");

        //public void write(int b) throws IOException
        fileOutputStream.write(97); //写入的是阿斯克码值，查表后得知，97 a

        //public void write(byte[] b) throws IOException
        byte[] b = {97, 98, 99, 100, 101, 102, 103};
        fileOutputStream.write(b);  //a b c d e f g

        //public void write(byte[] b,int off,int len) throws IOException
        fileOutputStream.write(b, 0, 5);//向后移0位，长度为5的，也就是 a b c d e

        //所以，我们可以得知，在b.txt 文件中写入的数据是
        //aabcdefgabcde

        fileOutputStream.close();
    }
}

成功，但为什么呢？

我们用FileInputStream读取数据到控制台时，我们发现数据乱码，但是为什么复制文件却没有问题。

因为在FileInputStream中我们认为的将字节转换了格式，所以会乱码，而这次我们没有做任何的转换，每读取一个字节，就会写一个字节，当读取两个字节时，系统会自动的将之转化问一个汉字。

代码如下：

public class CopyFile {
    public static void main(String[] args) throws IOException {

        FileOutputStream fileOutputStream = new FileOutputStream("d.txt");
        FileInputStream fileInputStream = new FileInputStream("b.txt");

        int by = 0;
        while ((by = fileInputStream.read()) != -1) {
            fileOutputStream.write(by);
        }

        fileInputStream.close();
        fileOutputStream.close();
    }
}

运行结果分析：

按照我们想的，一次五个字节，应该是第一次取到Ilove但是我们发现，他只取到了 lo ，为什么呢？

那是因为在windows操作系统下一个换行符是占了两个字节，换行符为 \r\n，所以我们发现他只取到了lo，着我们了解了，但是为什么在最后一次字节数组获取长度为-1，那是因为如果读入缓冲区的字节总数，如果因为已经到达文件末尾而没有更多的数据，则返回 -1。倒数第二次为hinaC，那是因为每次读取时会覆盖上一次读取的字符数组，但又因为最后一次，只读取了4个，所以没有完全覆盖上一次的字符，所以会出现这样的情况。

字节数组提高效率

七、字节缓冲流

出现原因：由上面我们知道，在利用字节数组进行大文件复制时，效率非常高，所以Java在设计的时候就考虑到了这个问题，所以就提供了，带缓冲区的字节类

这种类被称为缓冲区类（也叫高效类）

BufferedInputStream：字节缓冲输入流 ，读

BufferedOuputStream：字节缓冲输出流，写

public class MyBuffered {
    public static void main(String[] args)throws IOException {
        BufferedOutputStream bos = new BufferedOutputStream(new FileOutputStream("bb.txt"));
        bos.write("hello".getBytes());
        bos.close();
    }
}

九、总结

IO用于设备间进行数据传输的操作

IO分类：

  • 按流向分
    • 输入流：读取数据
    • 输出流：写出数据
  • 按数据类型分
    • 字节流
      • 字节输入流
      • 字节输出流
    • 字符流
      • 字符输入流
      • 字符输出流
  • 注意：
    1. 如果没有特殊要求，一般按数据类分
    2. 除非用Windows自带的记事本能打开，且能读懂，用字符流，否则就用字节流。
    3. 一般使用字节流

FileOutputStream写出数据（写入）：

  • 操作步骤
    • 创建字节输出流对象
    • 调用write();方法
    • 释放资源

FileInputStream（读取）：

  • 操作步骤
    • 创建字节输入流对象
    • 调用read()方法
    • 释放资源
