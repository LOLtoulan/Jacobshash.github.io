网络编程

一、网络编程的概述

计算机网络：是指在地理位置不同的具有独立功能的多台计算机及其外部设备，通过通信线路连接起来，在网络操作系统，网络管理软件网络管理软件及网络通信协议的管理和协调下，实现资源共享和信息传递的计算机系统。简单点说：就是在不同的计算机上进行相互通信的计算机系统

网络编程：就是用来实现网络互连的不同计算机上运行的程序键可以进行数据交换

详解TCP/IP

网络编程三要素：

  1. ip
  2. 端口
  3. 协议

举例说明：

看我之前博客的小伙伴都知道我女神是郑合惠子，我现在想和整合惠子说话，要怎么办呢？

首先，我要知道郑合惠子的地址（ip）

然后找到惠子的地址（ip）之后，我要去找她，通过地址（ip）我成功的找到了惠子，但是呢？

找到之后要进行交流，我总不可能对着脚说话吧，所以我要知道耳朵在哪里！也就是（端口号），

我们要进行交流，但是我说”俄语”，她有可能听不懂，那怎么办呢？然后我知道，她会汉语，我也会汉语，所以我们就协定（协议），我们用汉语交流。

ip是网络中计算机的唯一标识，但是我们知道计算机只能识别二进制数据，所以我知道，我们的IPV4地址在网络中其实就是一个二进制数据，但是，为什么不在一开始配置ip的时候就设置为二进制呢？那样不是就省省了很多事了吗？

  • 主要就是因为二进制数据不太容易记，而且，还容易记混，所以，为了方便IP的表示，我们把二进制转换为十进制，然后用”.”来进行分割，这种方法又称为”点分十进制”

  • 假设一个ip地址为：192.168.1.111那么我们用二进制方法来描述，那是多少呢？

    答案是：11000000 10101000 00000001 01101111 所以才有了”点分十进制”方法

协议

UDP（发短信）

    将数据员和目的地封装成数据包中,不需要建立连接；
    每个数据报的大小限制在64k；
    因为不连接，所以不可靠；
    不需要建立连接，速度快；

TCP（打电话）

    建立连接，形成传输数据的通道，在连接中可进行大量数据的传输；
    通过三次握手完成链接，是可靠协议；
    必须建立连接，效率会稍低；

举个例子（不太恰当）：

有一天，某地发生了一起火灾，用UDP协议进行灭火就是，有一个人，拿着水桶，一桶一桶的灭火，不需要向120进行保安，适合小型的火灾，而TCP则是，有一个人向120报案，120再三确认了目的地，然后开着救火车，来了，然后拿着水管进行灭火，这样我们就会发现，我们用水桶进行灭火时，我们的行动速度快，但是一次传输的水比较少，而我们用水管灭火就是，我们知道在灭火之前，我们必须等待消防车来，但是传输的水多。

Socket通信原理

Socket套接字：

  • 网络上具有唯一标识的IP和端口号组合在一起才能构成唯一能识别的的标识符套接字

Socket原理机制：

  • 通信的两端都有Socket
  • 网络通信就是Socket间的通信
  • 数据在两个Socket间通过IO传输

public class UDPSendDemo {

    public static void main(String[] args) throws IOException{
        //首先创建Socket，但是由于我们不是使用TCP协议所以我们不用Socket
        // 但是我们发送数据必须使用Socket，所以Java提供了UDP协议的Socket

        //创建发送端Socket对象
        DatagramSocket ds = new DatagramSocket();

        //创建数据并把数据打包

        // 发送数据包,因为要发送数据，按照Java命名特点，方法上应该有个send，
        // 所以我们在API中寻找该方法，我们发现真有这个方法，
        // public void send(DatagramPacket p)从此套接字发送数据报包。
        // DatagramPacket包括指示要发送的数据，其长度，远程主机的IP地址和远程主机上的端口号的信息。
        // 我们发现是DatagramPacket中类，所以我们又在API中进行寻找含有数据，长度，远程主机的IP地址和远程主机上的端口号
        // 的这么一个方法。最终我们发现了，如下
        // DatagramPacket(byte[] buf, int length, InetAddress address, int port)
        // 构造用于发送长度的分组的数据报包 length指定主机上到指定的端口号。

        //要发送的数据，因为是Byte[],so
        String s;
        byte[] bytes = "hello,UDP,我来了".getBytes();

        //长度
        int length = bytes.length;

        //远程主机IP
        InetAddress inetAddress = InetAddress.getByName("192.168.243.1");

        //端口号
        int port = 10088;

        DatagramPacket dp = new DatagramPacket(bytes, length,inetAddress,port);

        ds.send(dp);

        //释放资源
        ds.close();

    }
}
public class UDPReceiveDemo {
    public static void main(String[] args) throws IOException{

        //创建接收数据的对象
        DatagramSocket ds = new DatagramSocket(10088);

        //创建一个数据包，接收数据
        byte[] bytes = new byte[1024];

        //长度
        int length = bytes.length;

        DatagramPacket dp = new DatagramPacket(bytes, length);

        ds.receive(dp); //阻塞式方法

        InetAddress inetAddress = dp.getAddress();
        String ip = inetAddress.getHostAddress();

        byte[] bytes1 = dp.getData();
        int length1 = dp.getLength();
        String s = new String(bytes1, 0, length1);
        System.out.println(ip + ":" + s);
        //释放资源
        ds.close();
    }
}

多线程实现聊天室

代码如下：

public class ChatRoom {
    public static void main(String[] args) throws IOException {

        DatagramSocket sds = new DatagramSocket();

        DatagramSocket rds = new DatagramSocket(12306);

        SendThread st = new SendThread(sds);

        ReceiveThread rt = new ReceiveThread(rds);

        Thread t1 = new Thread(st);

        Thread t2 = new Thread(rt);

        t1.start();

        t2.start();
    }
}
public class ReceiveThread implements Runnable {

    private DatagramSocket ds;

    public ReceiveThread(DatagramSocket ds) {
        this.ds = ds;
    }

    @Override
    public void run() {
        try {
            while (true) {
                byte[] bytes = new byte[1024];

                DatagramPacket dp = new DatagramPacket(bytes, bytes.length);

                ds.receive(dp);

                String ip = dp.getAddress().getHostAddress();

                String s = new String(dp.getData(), 0, dp.getLength());

                System.out.println("from" + ip + " data is :" + s);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
public class SendThread implements Runnable {

    private DatagramSocket ds;

    public SendThread(DatagramSocket ds) {
        this.ds = ds;
    }

    @Override
    public void run() {
        try {
            BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

            String line = null;

            while ((line = br.readLine()) != null) {
                if ("886".equals(line)) {
                    break;
                }

                byte[] bytes = line.getBytes();

                DatagramPacket dp = new DatagramPacket(bytes, bytes.length, InetAddress.getByName("192.168.243.1"), 12306);

                ds.send(dp);
            }
            ds.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

/*
TCP协议接受资源
1.创建接收端的Socket对象
2.监听客户端连接，返回一个对应的Socket对象
3.获取输入流将数据显示在控制台上
4.释放资源
 */
public class TCPSeverDemo {
    public static void main(String[] args) throws IOException {

        //创建接收端Socket对象
        ServerSocket ss = new ServerSocket(8123);

        //监听客户端连接，返回一个对应的Socket对象
        Socket s = ss.accept();//阻塞方法

        //获取输入流，将数据显示在控制台上
        InputStream is = s.getInputStream();

        byte[] bytes = new byte[1024];
        int len = is.read(bytes);
        String str = new String(bytes, 0, len);

        String ip = s.getInetAddress().getHostAddress();
        System.out.println(ip+"----"+str);
        s.close();
    }
}
/*
TCP协议发送数据：
1.创建发送端的Socket对象
2.获取输出流，写数据
3.释放资源

我们如果不写服务端，我们这样运行这个程序时会被拒绝的，为什么？
这是因为，TCP协议进行通信需要经过三次握手，但是现在连服务器端都没有写，
根本就无法进行连接所以，会报 java.net.ConnectException: Connection refused: connect 异常

public class UserThrad implements Runnable{

    private Socket s = null;

    public UserThrad(Socket s) {
        this.s = s;
    }

    public UserThrad() {
    }

    @Override
    public void run() {
        try {
            BufferedInputStream bis = new BufferedInputStream(s.getInputStream());

            //防止文件名冲突
            String newName = System.currentTimeMillis() + ".jpg";

            BufferedOutputStream bos = new BufferedOutputStream(new FileOutputStream(newName));

            byte[] bytes = new byte[1024];

            int line = 0;

            while ((line = bis.read(bytes)) != -1) {

                bos.write(bytes, 0, line);
                bos.flush();

            }

            OutputStream os = s.getOutputStream();

            os.write("图片上传成功".getBytes());
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

五、总结

  1. 网络编程（用Java语言实现计算机间数据的传递和资源共享）
  2. 网络模型（OSI参考模型，TCP/IP参考模型）
  3. 网络编程三要素
    • ip
      • IP地址的组成
      • IP地址的分类
      • 点分十进制
      • dos命令（ipconfig）
      • InetAdderss（获得IP地址）
    • 端口
      • 是应用程序的标识，范围（065535），（01024为保留端口，不建议使用）
    • 协议
      • UDP：数据打包，每个最大64k，不进行连接，也就是不确保收到，效率高，但不可靠
      • TCP：不打包，建立数据通道，无限制，有连接，三次握手，四次挥手，效率低，可靠
  4. Socket机制
    • 通信两端都应该有Socket对象（UDP：DatagramSocket(发送)，DatagramSocket(接收)）（TCP：客户端(Socket)，服务器端(ServerSocket)）。
    • 所有操作都是通过Socket间的IO进行操作
  5. UDP发送和接收数据
    • 发送
      • 创建UDP发送端的Socket对象
      • 创建数据并把数据打包
      • 发送数据
      • 释放资源
    • 接受
      • 创建UDP接收端的Socket对象
      • 创建数据包用于接收数据
      • 接收数据
      • 解析数据包
      • 释放资源
  6. TCP协议发送和接收数据
    • 发送
      • 创建TCP客户端的Socket对象
      • 获取输出流，写数据
      • 释放资源
    • 接受
      • 创建TCP服务器端的Socket对象
      • 监听客户端连接
      • 获取输入流，读取数据
      • 释放资源
  7. 案例
    • UDP
      • 最基本的UDP协议发送和接收数据
      • 把发送数据改进为键盘录入
      • 一个简易聊天小程序并用多线程改进
    • TCP
      • 最基本的TCP协议发送和接收数据
      • 服务器给出反馈
      • 客户端键盘录入服务器控制台输出
      • 客户端键盘录入服务器写到文本文件
      • 客户端读取文本文件服务器控制台输出
      • 客户端读取文本文件服务器写到文本文件
      • 上传图片
      • 多线程改进上传文件
