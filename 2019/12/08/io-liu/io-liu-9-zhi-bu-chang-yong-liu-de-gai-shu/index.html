三、打印流

打印流的特点：

  • 只能操作目的地，不能操作数据。只能操作目的地，不能操作数据源
  • 可以操作任意类型的数据
  • 如果启用了自动刷新，则在调用println，printf，format的其中一个方法时，可以自动刷新
  • 可以操作文件的流

那些流对象可以直接操作文本文件

  • FileInputStream
  • FileOutputStream
  • FileReader
  • FileWriter
  • PrintStream
  • PrintWriter

如何判断一个类是否可以直接操作文件？

看API，查流对象的构造方法，如果同时有File类型和String类型的参数，一般来说是可以直接操作文件的

六、合并流

SequenceInputStream(Enumeration<? extends InputStream> e)初始化新创建 SequenceInputStream通过记住参数，它必须是一个 Enumeration产生对象，它们的运行时类型是 InputStream 。

SequenceInputStream(InputStream s1, InputStream s2) 通过记住两个 SequenceInputStream来初始化新创建的SequenceInputStream，这些参数将按顺序读取，首先是 s1 ，然后是 s2 ，以提供要从此 SequenceInputStream读取的字节。

/*

以前的复制
a.txt ---> b.txt

现在想要
a.txt + b.txt ---> c.txt

针对这一需求Java有提供了SequenceInputStream来解决
 */
public class SequenceInputStreamDemo {
    public static void main(String[] args) throws IOException {

        InputStream s1 = new FileInputStream("LOLtoulan\\a.txt");

        InputStream s2 = new FileInputStream("LOLtoulan\\b.txt");

        SequenceInputStream sis = new SequenceInputStream(s1, s2);

        BufferedOutputStream bos = new BufferedOutputStream(new FileOutputStream("LOLtoulan\\copy.txt"));

        byte[] bytes = new byte[1024];

        int len = 0;

        while ((len = sis.read(bytes)) != -1) {
            bos.write(bytes, 0, len);
        }
        bos.close();
        sis.close();
    }
}
/*

以前的复制
a.txt ---> b.txt

现在想要
a.txt + b.txt ---> c.txt

甲方又想了，既然2个文件可以复制到一个文件中，那多于两个呢？
a.txt + b.txt + c.txt ---> d.txt


针对这一需求Java有提供了SequenceInputStream中的SequenceInputStream(Enumeration<? extends InputStream> e) 来解决
 */
public class SequenceInputStreamDemo2 {
    public static void main(String[] args) throws IOException {
        Vector<InputStream> vector = new Vector<>();

        InputStream s1 = new FileInputStream("LOLtoulan\\a.txt");

        InputStream s2 = new FileInputStream("LOLtoulan\\b.txt");

        InputStream s3 = new FileInputStream("LOLtoulan\\copy.txt");

        vector.addElement(s1);

        vector.addElement(s2);

        vector.addElement(s3);

        Enumeration<InputStream> elements = vector.elements();

        SequenceInputStream sis = new SequenceInputStream(elements);

        BufferedOutputStream bos = new BufferedOutputStream(new FileOutputStream("LOLtoulan\\Copy3.txt"));

        byte[] bytes = new byte[1024];

        int len = 0;

        while ((len = sis.read(bytes)) != -1) {
            bos.write(bytes, 0, len);
        }
        bos.close();
        sis.close();
    }
}

七、序列化流和反序列化流的概述和使用

序列化流：把对象按照流一样的方式存入文本文件或者在网络中传输。 对象 —– 流数据 （ObjectOutputStream）

反序列化流：把文本文件中的流对象数据或者网络中的流对象数据还原成对象 。流数据 ——-对象 （ObjectInputStream）

Student.java

/**
 * java.io.NotSerializableException
 * 如果此处不实现序列化接口，会报没有序列化异常，
 * 而且我们查看API时发现，该接口中竟然没有方法，在Java中类似这种没有方法的接口
 * 叫做标记接口
 */
public class Student implements Serializable {
    private String name;
    private int age;

    public Student() {
    }

    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    @Override
    public String toString() {
        return "Student{" +
                "name='" + name + '\'' +
                ", age=" + age +
                '}';
    }
}

public class ObjectOutputStreamDemo {
    public static void main(String[] args) throws IOException, ClassNotFoundException {

        //write();

    }

        ObjectInputStream ois = new ObjectInputStream(new FileInputStream("Student.txt"));

        System.out.println(obj);
        System.out.println(obj1);

        ois.close();
    }

    private static void write() throws IOException {

        //创建序列化流对象
        ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("Student.txt"));

        Student s1 = new Student("桥本环奈", 19);

        //将对象写道序列化流创建的文件中
        oos.writeObject(s);
        oos.writeObject(s1);

        //刷新缓存
        oos.flush();

        //关闭流对象
        oos.close();
    }
}

java.io.InvalidClassException: MyObjectOutputStream.Student; local class incompatible: stream classdesc serialVersionUID = -7394093041140435532, local class serialVersionUID = -2897294725402607106

当我们查API的时候，我们发现是一个序列化异常。

当序列化检测到以下之一时抛出：

  • 类的串行版本与从流中读取的类描述符的类型不匹配
  • 该类包含未知的数据类型
  • 该类没有可访问的无参数构造函数

出现问题的原因是:

当我们的Student类实现序列化接口时，因为Serializable是一个标记接口，所以，当程序运行时，回得到一个值，我们暂定为id，值暂定为100，当程序首次运行时

Student.calss —– id = 100;

student.txt 收到的id 为 100；

studnet .txt 第一次读取时 id =100

当我们修改了Student.java时程序发现现在的类不是原来的那个类，所以重新给了 一个新的id ,假定值为101，这时，因为我们之操作读数据，此时的Student.txt 中的id 还是100，不是同一个id，所以就抛了一个异常，我们刚才的情况刚好符合该异常的第一条。

错误我知道了，所以下一步就要修改，那就是重新写入数据，重新读取数据就可以了。

但是我们在开发中，有可能原来的数据还要继续使用，如果直接重新写入，有可能会导致数据的丢失，这样损失是巨大的，所以我们需要另一种方法，来进行操作。

我们知道，我们无法读取的原因是id值得不一样，所以如果我们有办法让两个值相等那么我们就可以直接进行读取，而且我们发现在报错信息中，非常明显的一点，是serialVersionUID 的不同，那么我们就设置serialVersionUID 为私有静态长整型，这样serialVersionUID 就是一样的了，因为静态数据只会执行一次,

值得注意的是，这个 serialVersionUID 的值为stream classdesc serialVersionUID = -7394093041140435532，否则就不会读取成功

八、Properties的概述

由API我们知道Properties类表示了一个持久的属性集，Properties可以保存在流中，或者从流中加载属性列表每个键其对应值都是一个字符串。

因为Properties从继承Hashtable时， put种putAll方法可应用于Properties对象。

我们知道，Properties是继承于Hashtable<Object,Object>，所以我们可以利用HashTable的方法来学习该类。

public class MyProperties {
    public static void main(String[] args) {
        Properties properties = new Properties();

        properties.put("789", "qwe");
        properties.put("78", "qwe");
        properties.put("7", "qwe");


        Set<Object> setKey = properties.keySet();
        for (Object o : setKey) {
            Object value = properties.get(o);
            System.out.println(o +"----"+value);
        }
    }
}

当我们再次查看API的时候发现Properties不是一个泛型类也就是说该类不存在（Properties<?>）这种形式，所以我们知道该类是一个Map类型的集合，但是键值对可能只能有一种类型，于是我们查看API发现下图，

所以我们知道，该集合只能传String类型的键和值，所以官方文档才不推荐我们直接调用put()方法来进行赋值操作因为这样做是不安全的，那么，到底是怎样做到的，只能传String的方法呢？

我们查看源码如下：

public class Properties extends Hashtable<Object,Object> {
     //构造方法
    public Properties() {
        this(null);
    }

    public Properties(Properties defaults) {
        this.defaults = defaults;
    }

    //setProperty方法
    public synchronized Object setProperty(String key, String value) {
        return put(key, value);
    }
}

我们发现在该方法中，该方法新写一个方法，来接受properties传过来的数据，且该方法的参数是String类型的，虽然是String类型的，但是我们发现，该方法，底层调用的还是HashTable的put方法，所以，我们知道其实归根结底，还是调用的put方法，下面我们来写一个小案例

public class MyProperties {
    public static void main(String[] args) {
        Properties properties = new Properties();

        properties.setProperty("qwe", "qwe");
        properties.setProperty("桥本环奈", "21");
        properties.setProperty("斋藤飞鸟", "22");
        properties.setProperty("郑合惠子", "20");

        //因为是K V类型的集合，所以可以用遍历Map的方法遍历Properties类
        Set<Object> setKey = properties.keySet();
        for (Object o : setKey) {
            Object value = properties.get(o);
            System.out.println(o +"----"+value);
        }

        //Properties特有的遍历方法
        //获取所有键的集合
        Set<String> sets = properties.stringPropertyNames();
        for (String s : sets) {
            //通过键寻值
            Object value = properties.getProperty(s);
            System.out.println(s +"----"+value);
        }
    }
}

    }
    //字符数组
    char[] lineBuf = new char[1024];
    //对进行分割
    private void load0 (LineReader lr) throws IOException {
        char[] convtBuf = new char[1024];
        int limit;
        int keyLen;
        int valueStart;
        char c;
        boolean hasSep;
        boolean precedingBackslash;

        //和我们自己写的一样，读取文件中的数据
        while ((limit = lr.readLine()) >= 0) {
            c = 0;
            keyLen = 0;
            valueStart = limit;
            hasSep = false;

            //System.out.println("line=<" + new String(lineBuf, 0, limit) + ">");
            precedingBackslash = false;
            while (keyLen < limit) {
                //一次读取一个字节数组的长度
                c = lr.lineBuf[keyLen];
                //need check if escaped.
                //检查是否需要转译
                if ((c == '=' ||  c == ':') && !precedingBackslash) {
                    valueStart = keyLen + 1;
                    hasSep = true;
                    //跳出这个while循环执行外面的while循环
                    break;
                } else if ((c == ' ' || c == '\t' ||  c == '\f') && !precedingBackslash) {
                    valueStart = keyLen + 1;
                    //跳出这个while循环执行外面的while循环
                    break;
                }
                if (c == '\\') {
                    precedingBackslash = !precedingBackslash;
                } else {
                    precedingBackslash = false;
                }
                keyLen++;
            }
            while (valueStart < limit) {
                c = lr.lineBuf[valueStart];
                if (c != ' ' && c != '\t' &&  c != '\f') {
                    if (!hasSep && (c == '=' ||  c == ':')) {
                        hasSep = true;
                    } else {
                        break;
                    }
                }
                valueStart++;
            }
            String key = loadConvert(lr.lineBuf, 0, keyLen, convtBuf);
            String value = loadConvert(lr.lineBuf, valueStart, limit - valueStart, convtBuf);
            //将数据put进集合中
            put(key, value);
        }
    }
}

有几何我们知道我们之所以能够只调用一个方法就将里面的数据推到集合中，是因为Java底层已经做好了，底层做好就会有利有弊，有源码中我们发现，该方法只识别”=”和”:”，也就是说，如果在我们的文件中出现了其他的分割符系统是识别不了的，所以我们要理性看待源码。当然，大多数时间我们用到的分隔符就是这两个。

判断集合中是否有对应的键，有就修改

该题分四步：

  • 将文件中的数据放入集合中
  • 遍历集合
  • 将数据修改
  • 将数据写入文件中

public class MyProperties3 {
    public static void main(String[] args) throws IOException {

        Properties properties = new Properties();
        Reader reader = new FileReader("LOLtoulan\\xyz1.properties");
        properties.load(reader);

        Set<String> sets = properties.stringPropertyNames();
        for (String s : sets) {
            if (s.equals("qwe")) {
                String value = "我被修改了";
                properties.setProperty(s, value);
            }
        }
         //这里其实是将集合中遍历的数据又存到集合中了
        properties.store(new FileWriter("LOLtoulan\\xyz1.properties"),"xiugail");
        reader.close();
        Set<Object> setKey = properties.keySet();
        for (Object o : setKey) {
            Object value = properties.get(o);
            System.out.println(o +"----"+value);
        }
    }

九、总结

数据操作流：

  • 可以操作基本数据类型的流
  • 流对象名称
    • DataInputStream
    • DataOutputStream

内存操作流

  • ByteArrayInputStream，ByteArrayOutputStream
  • CharArrayReader，CharArrayWriter
  • StringReader，StringWriter

打印流

  • 字节打印流，字符打印流
  • 特点：
    • 只操作目的地，不操作数据源，
    • 可以操作任意类型的数据
    • 如果启用了自动刷新，那么在调用Println()方法时，能够换行并刷新
    • 可以直接操作文件

标准输入输出流

  • 可以按照文件指针的位置进行读写数据

合并流

  • 把多个输入流语句写到输出流中

序列化流

  • 可以把对象写入文本文件中或者在网络中传输
  • 如何实现序列化？
    • 让被序列化的对象所属的类实现序列化接口
    • 该接口是标记接口没有功能需要实现
  • 注意问题：
    • 把文件写入数据后再去修改类会出现一个异常InvalidClassException
    • 这是我们就要把实现序列化接口的文件给一个固定的序列化id值。
    • private static final long serialVersionUID = xxx;
  • 常见面试题
    • 什么是序列化，什么是反序列化，如何实现序列化

Properties

  • properties.setProperty();
  • properties.getProperty();
  • properties.propertyNames();
  • public void store(OutputStream out,String comments); //把集合中的文件存储到文件
  • public void load(InputStream inStream); //把文件中的数据读取到集合中
