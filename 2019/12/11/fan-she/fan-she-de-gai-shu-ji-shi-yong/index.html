反射的概述及使用

一、类的加载

类的加载

  • 当程序要使用某个类时，如果该类还没有被加载到内存中，则系统会通过加载，连接，初始化三步来实现对这个类进行初始化

  • 加载

    • 就是指将class文件读入内存，并为之创建一个Class对象。

      Class：是Object类中的一个方法，方法如下：

      public final Class<?> getClass()
      返回此Object的运行时类。 返回的类对象是被表示类的static synchronized方法锁定的对象。

      一个Class文件基本由三个部分组成：

      成员变量，构造方法，成员方法

    • 任何类被使用时都会创建一个Class对象。

  • 连接

    • 验证：是否有正确的内部结构，并和其他类协调一致
    • 准备：负责为类的静态成员分配内存并设置默认初始化值
    • 解析：将类的二进制数据中的符号引用替换为直接引用

  • 初始化

    • 为栈开辟空间
    • 为堆开辟空间
    • 默认初始化
    • 闲置初始化
    • 构造初始化
    • 对类的静态变量，静态代码块执行初始化操作

类初始化时机

  • 创建类的实例
  • 访问类的静态变量，或为静态变量赋值
  • 调用类的静态方法
  • 使用反射方式类似强制创建某个类，或接口对应的java.lang.Class对象
  • 初始化某个类的子类
  • 直接使用java.exe命令来运行某个主类
类加载器

类加载器

  • 负责将.class文件加载到内存中，并为之生成对应的class对象
  • 虽然我们不需要关心类加载机制，但是更好的了解这个机制，我们就能更好的理解程序的运行

类加载器组成

  • Bootstrap ClassLoader 根类加载器
  • Extension ClassLoader 扩展类加载器
  • System ClassLoader 系统类加载器
类加载器的作用

Bootstrap ClassLoader 根类加载器

  • 亦被称为引导类加载器，负责Java核心类的加载
    • 比如System，String等。在jdk中的jre目录下的rt.jar文件中

Extension ClassLoader 扩展类加载器

  • 负责JRE的扩展目录中jar包的加载。
    • 在JDK中JRE的lib目录下的ext目录

System ClassLoader 系统类加载器（也就是加载我们写的文件）

  • 负责在jvm启动时加载来自Java命令的class文件，以及classpath环境变量所指定的jar包和类路径
    通过这些描述我们就可以知道我们常用的东西的加载都是由谁来完成的。
    到目前为止我们已经知道把class文件加载到内存了，那么，如果我们仅仅站在这些class文件的角度，我们如何来使    用这些class文件中的内容呢?
    这就是我们反射要研究的内容。

二、反射的概述

Java反射机制是在运行状态中，对于任意一个类，都能够知道这个类的所有属性和方法；对于任意一个对象，都能够调用他的任意一个方法及属性；这种动态获取的信息以及动态调用对象的方法成为Java语言的反射机制。

要想解刨一个类，必须先要获取到该类的字节码文件对象。而解刨使用的就是Class类中的方法，所以要先获取每一个字节码对应的Class类型对象。

如何获取呢？

有三种方法：

  1. Object类的getClass方法

  2. 数据类型的静态属性Class

  3. Class类中的静态方法

    public static Class forName(String className) ;

示例代码如下：

/*
反射就是通过class文件对象，去使用该类中的成员变量，构造方法，成员方法
 */
public class ReflectDemo1 {
    public static void main(String[] args) throws ClassNotFoundException {
        //第一种方式
        Person p1 = new Person();
        Class c1 = p1.getClass();

        Person p2 = new Person();
        Class c2 = p2.getClass();

        System.out.println(p1 == p2); //false
        System.out.println(c1 == c2); //true

        //第二种
        Class c3 = Person.class;
        System.out.println(c1 == c2 && c2== c3 && c1 ==c3); //true

        //第三种
        //Exception in thread "main" java.lang.ClassNotFoundException: Person
        //这里必须写全路径，否则会报错
        Class c4 = Class.forName("MyDemo1.Person");
        System.out.println(c4 == c1 );   //true
    }
}

public class ReflectDemo {

        Class c = Class.forName("MyDemo1.Person");

        //获取构造方法

        Constructor[] cons = c.getConstructors();
        for (Constructor con : cons) {
            System.out.println(con);
        }

        System.out.println("------------------------");

        Constructor[] dcons = c.getDeclaredConstructors();
        for (Constructor con : dcons) {
            System.out.println(con);
        }

        System.out.println("------------------------");
        //虽然有很多的构造方法 ，但是我们在开发的时候，有时只需要一个就可以了，所以java提供了一种可以指获取一个构造参数的方法
        //public Constructor<T> getConstructor(Class<?>... parameterTypes)
        //该方法的参数是可变参数，也就是我们可以传0~多个参数，该方法会根据我们传的参数来给出我们想要的构造器

        //public T newInstance(Object... initargs)
        //使用此Constructor 对象的构造方法来创建改构造方法的新实例，并用指定的初始化参数，初始化该实例

        Constructor con = c.getConstructor();
        System.out.println(con);
        Object obj = con.newInstance();
        System.out.println(obj);

        //上面的三步操作，其实相当于
        //Person p = new Person();
        //System.out.println(p);
        // 但是我们发现我们并没有导Person的包，
        // 反射就是不会让我们在程序中找到任何关于我们运行类相关的代码，

    }

/*
通过反射获取成员变量并使用

public class ReflectDemo {
    public static void main(String[] args) throws Exception {

        Class c = Class.forName("MyDemo1.Person");

        Constructor dc = c.getDeclaredConstructor();

        Object o = dc.newInstance();


        Method[] methods = c.getMethods();  //获取所有的公共方法（包括父亲的公共方法）
        for (Method m : methods) {
            System.out.println(m);
        }

        System.out.println("------------------------");

        Method[] dms = c.getDeclaredMethods();  //获取自己的所有方法（包括私有）
        for (Method m : dms) {
            System.out.println(m);
        }

        System.out.println("------------------------");

        //public Method getMethod(String name,Class<?>... parameterTypes)
        //第一个参数是方法名，第二个参数是方法参数的class类型。
        //例如：Method show = c.getMethod("getString",String.class,int.class);

        Method show = c.getMethod("show");

        //在具有指定参数的方法对象上调用此方法对象表示的基础方法。
        // 个别参数自动解包以匹配原始形式参数，原始参考参数和参考参数都需要进行方法调用转换。
        show.invoke(o);

        Method function = c.getDeclaredMethod("function");
        function.setAccessible(true);
        function.invoke(o);

        //public Object invoke(Object obj,Object... args)
        // 返回值是Object接受，第一个是指对象是谁，第二个是指实际参数传什么值
        Method method = c.getMethod("method", String.class);
        method.setAccessible(true);
        method.invoke(o, "haha");

        Method getString = c.getMethod("getString", String.class, int.class);
        getString.setAccessible(true);
        Object getstring = getString.invoke(o, "我是getString方法", 21);
        System.out.println(getstring);
    }
}

通过反射越过泛型检查并使用

按道理来说，我们在写ArrayList时，我们会将他的泛型给加上，因为加上泛型之后，程序更加安全了，

比如说，我们在进行添加数据时，如果我们这样定义ArrayList arrayList = new ArrayList();

那么在添加元素时，我们无论添加任何元素都是正确的，也就是不会报错

但是如果我们这样定义：ArrayList<Integer> arrayList = new ArrayList<>();,我们在添加元素时我们就只能添加Integer类型的元素，那么，这是为什么呢？

因为我们这一但进行限定集合的添加属性之后，编译器就会在在编译的时候进行类型限定，这样，我们就会在，添加元素的时候更加清楚。

也就是说，增强了代码的可读性。

但是，我就不想走寻常路，我就想在ArrayList<Integer> arrayList = new ArrayList<>();中添加String类型的字符串，可以吗？

答案是：可以的，当然这离不开，我们本篇博客所讲的反射，当然则要看源码。

我们明确知道，集合中有添加功能，也就是add()方法，所以我们今天就看看如何通过反射，越过泛型检查，我们先看一段代码，

public class ReflectDemo {

        arrayList.add(12);
        arrayList.add(123);
        //arrayList.add("4646");  //报错
        System.out.println(arrayList);
    }
}

首先，我们发现ArrayList的泛型标识不见了，这说明泛型只会在编译器有作用，也就是，泛型是给编译器和程序员看的，真正运行期是不用泛型标识的。接下来，我们看看集合的add()方法是如何定义的，

通过源码我们发现，其实我们在进行add()方法进行添加元素的时候，add()方法其实是无法判断传过来的元素是什么类型的，通过这里我们就发现，其实不管ArrayList如何设置泛型，其实只要我们越过编译器，直接使用ArrayList的class文件进行添加不就可以了吗？其实就是这样来做的。

/*

通过反射越过泛型检查

 */
public class ReflectDemo {
    public static void main(String[] args) throws Exception {
        ArrayList<Integer> arrayList = new ArrayList();

        arrayList.add(12);
        arrayList.add(123);
        //arrayList.add("4646");  //报错


        //反射三种方法之第一种，创建反射字节码对象
        Class c = arrayList.getClass();

        //通过反射字节码对象，得到ArrayList中的add方法
        Method method = c.getMethod("add", Object.class);

        //通过invoke方法给ArrayList添加元素
        method.invoke(arrayList, "我是测试");

        //打印元素
        System.out.println(arrayList);
    }
}

反射应用举例

写一个方法，

public void setProperty(Object obj, String propertyName, Object value){}，

此方法可将obj对象中名为propertyName的属性的值设置为value。

public class Tool {

    //此方法可将obj对象中名为propertyName的属性的值设置为value
    public void setProperty(Object obj, String propertyName, Object value) throws NoSuchFieldException, IllegalAccessException {

        //获取字节码文件对象
        Class c = obj.getClass();

        //通过字节码对象获取propertyName成员变量
        Field field = c.getDeclaredField(propertyName);

        //取消访问检查
        field.setAccessible(true);

        //给对象的成员变量设置为指定值
        field.set(obj, value);
    }
}

动态代理

就好比你去租房子时我们一般不会自己去找房东，而只找中介，让中介帮忙寻找，我们就是住进去而已，动态代理也是这个原理

动态代理
代理：本来应该自己做的事情，却请了别人来做，被请的人就是代理对象。
举例：春季回家买票让人代买
动态代理：在程序运行过程中产生的这个对象
而程序运行过程中产生对象其实就是我们刚才反射讲解的内容，所以，动态代理其实就是通过反射来生成一个代理

代码如下：

public class MyInvocationHandler implements InvocationHandler {

    private Object target;

    public MyInvocationHandler(Object target) {
        this.target = target;
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {

        System.out.println("权限校验");
        //target传过来的参数，也就是我们实例化的对象
        Object result = method.invoke(target, args);
        System.out.println("日志记录");

        return result;
    }
}
